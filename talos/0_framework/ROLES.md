# Roles: The Carbon-Silicon Divide

> **"Render unto the Agent the things that are Algorithm; render unto the Human the things that are Meaning."**

Talos is a **Human-Centric Company**. We do not replace humans; we liberate them from drudgery to focus on what matters.

## 1. The Separation of Concerns

| Attribute | The Human (Carbon) | The Agent (Silicon) |
| :--- | :--- | :--- |
| **Primary Domain** | Purpose, Ethics, Taste, Novelty. | Scale, Logic, Persistence, Data. |
| **The "Job"** | The Captain (Setting the Course). | The Crew (Rowing the Ship). |
| **Metric** | **High-Leverage Time** (Strategy/Creativity). | **Throughput** (Tasks Completed/Hour). |

## 2. The Human Roles

### A. The Strategist
*   **Responsibility**: Defining the *Why*.
*   **Action**: Setting the moral compass and the strategic objectives.
*   **Input**: Human values, market needs.
*   **Output**: Clear Directives (`talos/1_strategy/MANIFESTO.md`).

### B. The Guardian (Taste)
*   **Responsibility**: Ensuring Excellence.
*   **Action**: Rejecting "average" work. A Talos output must feel "crafted," not generated.
*   **Gate**: The Human is the final barrier between the machine and the world.

### C. The Engineer
*   **Responsibility**: Building the Machine (Talos itself).
*   **Action**: Debugging the *framework*, not the task. If an agent fails, you don't fix the task; you fix the `SKILL.md` or the `PROMPT`.

## 3. The Agent Roles

### A. The Architect
*   **Responsibility**: Planning.
*   **Input**: Human Strategy.
*   **Output**: Step-by-step YAML plans.

### B. The Builder
*   **Responsibility**: Execution.
*   **Input**: Blueprints.
*   **Output**: Code/Content.

### C. The Devil's Advocate (Silicon Critic)
*   **Responsibility**: Pre-Human Filtering.
*   **Action**: aggressively finding flaws *before* the human sees it.
*   **Goal**: Save the human from reviewing garbage.

## 4. The Golden Rule of Interaction
**"Never fix the output; fix the machine."**
If an agent produces bad code, do not rewrite it yourself. Update the `SKILL.md`, the `MANIFESTO.md`, or the `Context`. If you fix it manually, you are not building a company; you are just doing the work yourself.
